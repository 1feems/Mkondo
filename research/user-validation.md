# FlowOS — User Flow and Asset

**Purpose:** FlowOS was mapped against the Stacks grant program to confirm the product fits how a real program operates. This documents what was tested, what works, and where FlowOS fits in the workflow for the people running it.

---

## What Was Tested

- **Program Info Form** — tested against Stacks program-level setup: ecosystem identity, eligibility criteria, outcome focus, vertical, and expected deliverables
- **Round Intent Form** — tested against Stacks round-level setup: grant type, budget, target applicants, project types in scope, and success metrics
- **Pre-Screening** — tested against Stacks eligibility requirements: three questions map cleanly to program criteria and outcome focus
- **Rubric** — tested against Stacks evaluation criteria: FlowOS's six criteria match Stacks exactly — same names, same 0–5 scale
- **Application Form** — BNS One (an approved application) used as a reference to confirm the form captures what reviewers need

## What Validated

- Rubric criteria and scoring scale matched without modification
- Pre-screening maps cleanly to Stacks eligibility requirements
- Form fields cover what reviewers need to evaluate strategic alignment, feasibility, ecosystem impact, and budget reasonableness
- Registry KPI fields align with the outcome metrics Stacks tracks post-approval

---

## Where FlowOS Fits

**Grant Operators** — the team running the program day to day

| Step | FlowOS |
|---|---|
| Define what the program funds | Program setup forms — eligibility, rubric, outcome focus, and grant scope defined once and applied every round |
| Open a new round | Round setup — budget, who the round is for, and how success is measured |
| Receive applications | AI screening labels every submission before a reviewer sees it — Pass / Needs Revision / Not a Fit |
| Run review | Every application arrives with rubric scores already surfaced — reviewers work from the same criteria every round |

**Treasury Committee** — meets quarterly to review what the grant program produced and approve strategy and budgets

| Step | FlowOS |
|---|---|
| Quarterly review calls | The registry gives the committee a structured record per round — funded projects, outcome metrics, and what each round produced — ready to pull from before every call |
| Budget approvals | Program setup creates a record of what each program is designed to fund, at what cost, and against what metrics — decisions are grounded in structured data, not recall |

---

*Tested against: Stacks Endowment live application form, eligibility and evaluation guide, Builder Grant program track, and approved BNS One application (SIP-31 #31).*
